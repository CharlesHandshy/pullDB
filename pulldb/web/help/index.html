Try It Now

See pullDB in action with these interactive demos

pulldb restore
# Submit a restore job for customer "acme"
$ pulldb restore acme
Job queued successfully!
customer: acme
target: chrlesacme
staging_name: chrlesacme7f3a2b1c
job_id: 7f3a2b1c-85a1-4da2-...
status: queued
user: charles (chrles)
Use 'pulldb status' to monitor progress.

What's happening?

  1. pullDB searches S3 for the latest backup matching "acme"
  2. A job is created with your user code prefix (chrles)
  3. The job enters the queue for processing
  4. Worker downloads, extracts, and restores the backup
Learn more about restore options →
pulldb status
# Check status of your last job
$ pulldb status
Your last submitted job:
STATUS : running
OPERATION: Restore progress
JOB_ID : 7f3a2b1c
CUSTOMER : acme
TARGET : chrlesacme
DB : localhost
USER : chrles
SUBMITTED: 2026-01-03T10:42:15
STARTED : 2026-01-03T10:42:18

Status Information

  • STATUS: Current job state (queued, running, complete, failed, canceled)
  • OPERATION: What the job is currently doing
  • Use --rt: Stream events in realtime with pulldb status --rt
View all status options →
pulldb cancel
# Cancel a running job
$ pulldb cancel 7f3a2b1c
Requesting cancellation for job: 7f3a2b1c...
Are you sure you want to cancel this job? [y/N]: y
✓ Job 7f3a2b1c... canceled successfully.
Cancellation complete

Cancellation Notes

  • Cancellation is graceful—no data corruption
  • Staging databases are automatically cleaned up
  • Downloaded files are removed from disk
