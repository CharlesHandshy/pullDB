Dashboard

The Dashboard is your home base in pullDB. It provides an at-a-glance view of your jobs, system status, and quick actions tailored to your role.

Overview

What you see on the Dashboard depends on your role. Users with the user role see their own job activity, manager users see system-wide metrics, and admin users get full access to administration panels.

Dashboard features available by user role
Role Dashboard Features
user Recent jobs, quick restore, job stats for your account
manager All user features + system-wide job overview, Manager View link
admin All manager features + admin panel, system health, user management

User Dashboard

The standard user dashboard focuses on your personal job activity and provides quick access to common actions.

User dashboard layout
The user dashboard shows stats cards at the top (active jobs, last job status), followed by the Recent Jobs panel, and Quick Action buttons below.

Recent Jobs Panel

Shows your 5 most recent restore jobs with current status. Each job row displays:

  • Target Database — The restored database name
  • Status Badge — Current job state (Queued, Running, Complete, Failed)
  • Submitted Time — When you submitted the job
  • Actions — View details, cancel (if applicable)
Real-time updates
Job statuses update automatically via polling. No need to refresh the page.

Quick Restore Card

Start a new restore directly from the dashboard. Enter a customer ID and optionally select a specific backup date to quickly submit a job.

Manager View

Managers have access to the Manager View, accessible from the sidebar. This provides a comprehensive overview of all jobs across the system.

Manager dashboard layout
The manager dashboard extends the user view with a Team Members table and team-wide statistics below the standard panels.

Features

  • Team Members Table — Lists each managed user with their active job count and recent job status
  • Team Statistics — Total active jobs across the team, total databases currently in use
  • Team Recent Jobs — Last 5 jobs across the whole team
  • View Team Member Jobs — Click a team member row to see their individual job list
Manager permissions
Managers can view all team member jobs and can cancel jobs belonging to users they manage. They can also submit restore jobs on behalf of managed users.

Admin Dashboard

Admins see additional panels and have quick access to administrative functions.

Admin dashboard layout
The admin dashboard extends the user view with system-wide statistics, a Host Health table showing active restore counts per host, and the last 10 jobs across the whole system.

System Health Panel

Real-time system status including:

  • Worker Status — Is the worker service running?
  • Queue Depth — Number of jobs waiting to be processed
  • Active Jobs — Currently running restore operations
  • Disk Usage — Available space on the restore target

Admin Quick Links

One-click access to frequently used admin functions:

Manage Users

Add, edit, or disable user accounts.

Manage Hosts

Configure allowed restore targets.

API Keys

View and manage all API keys.

Settings

System configuration and preferences.

Stats Cards

The top of the dashboard displays summary statistics in card format.

Total Jobs

Total number of jobs you've submitted (or system-wide for managers/admins).

In Progress

Jobs currently queued or running.

Complete

Successfully restored databases (deployed, complete, expired).

Failed

Jobs that encountered errors.

Quick Actions

The dashboard provides quick action buttons for common tasks:

Action Description Roles
New Restore Opens the restore workflow page All
View All Jobs Navigate to full job list All
Refresh Manually refresh dashboard data All
View History Navigate to job history (terminal-state jobs) All
