Restore Workflow

The restore page provides a guided wizard for submitting database restore jobs. Follow the steps to select a customer, choose a backup, configure options, and submit.

~2 minutes to submit

Overview

The restore workflow is a 4-step wizard that guides you through submitting a restore job. Each step validates your input before allowing you to proceed.

Step 1: Select Source

Choose what to restore: a customer database or the QA template. Use the tabs at the top of the form to switch between modes.

Restore Mode Tabs

Customer

Restore a specific customer's database from their backup.

QA Template

Restore the shared QA template database for testing.

Customer Mode

Enter the customer ID whose database you want to restore:

  • Auto-complete — Start typing to see matching customer IDs
  • Validation — The system checks if backups exist before proceeding
  • Recent Customers — Quick links to customers you've restored recently
Case Sensitivity
Customer IDs are case-sensitive. Make sure to enter them exactly as they appear in your backup system.

QA Template Mode

The QA template is a pre-sanitized database for testing and development. It contains representative data without sensitive customer information.

  • No Customer Selection — Skip straight to backup selection
  • Pre-Sanitized Data — No post-restore scripts needed
  • Target Name — Always restores to {user_code}qatemplate

What Happens

When you enter a customer ID and click "Next", the system queries S3 for available backups. If no backups are found, you'll see an error message and won't be able to proceed.

Step 2: Choose Backup

Select which backup to restore from the list of available backups.

Backup List

The backup list shows all available backups for the selected customer, with the most recent first. Each backup entry displays:

Column Description
Date Backup date (YYYY-MM-DD)
Time Backup time in UTC
Size Compressed backup size
Environment Source environment (prod, staging, etc.)
Default Selection
The most recent backup is pre-selected. Click on any row to select a different backup.

Step 3: Configure Options

Configure advanced options for the restore job. All options have sensible defaults, so you can skip this step for standard restores.

Target Database Name

The target database name is displayed in the preview callout once you select a backup. By default, names are auto-generated:

Mode Format Example
Customer (no suffix) {user_code}{customer} jsmithacmecorp
Customer (with suffix) {user_code}{customer}{suffix} jsmithacmecorpdev
QA Template {user_code}qatemplate jsmithqatemplate

Custom Target Name

Click the "Custom" checkbox to override the auto-generated name with your own:

  • Pattern — Lowercase letters only (a-z)
  • Length — 1 to 51 characters maximum
  • Validation — Real-time feedback on invalid characters
No Suffix with Custom Target
When using a custom target name, the suffix field is hidden. Include any suffix directly in your custom name (e.g., myprojectdev).

Available Options

Option Description Default
Target Host MySQL server to restore to Default configured host
Suffix 1-3 letter suffix appended to database name None
Custom Toggle custom target name input Off (auto-generated)
Overwrite Drop database if it already exists Off
Overwrite Warning
When "Overwrite" is enabled, any existing database with the target name will be permanently deleted. Make sure you don't have important data in the target database.

Target Host Selection

The target host dropdown shows all hosts you're authorized to restore to. If you only have access to one host, it will be pre-selected and the dropdown disabled.

Step 4: Review & Confirm

Review all your selections before submitting the job. This step shows a summary of what will happen when you click "Submit".

Summary Panel

The confirmation page displays:

  • Customer ID — The customer being restored
  • Backup Date — Which backup will be used
  • Target Database — Where the data will be restored
  • Options Summary — All configured options
  • Estimated Time — Approximate duration based on backup size

Submit Button

Click Submit Restore Job to queue the job. You'll be redirected to the job details page where you can monitor progress.

Go Back
Use the "Back" button or click on any completed step number to go back and change your selections.

After Submission

Once you submit the job, you'll be taken to the job details page. Here's what to expect:

1. QUEUED

Job is waiting for a worker. Usually takes only a few seconds.

2. RUNNING

Worker is processing. Progress bar shows current phase (Download → Restore → Post-SQL).

3. COMPLETE

Database is ready! Connection details shown on completion.

Real-Time Updates

The job details page polls for updates automatically. You'll see:

  • Progress Bar — Visual progress through phases
  • Phase Indicator — Current processing phase name
  • Time Elapsed — How long the job has been running
  • Log Stream — Live log output (expandable)
