Manager Interface

The Manager page provides tools for monitoring your team members' activity and managing their restore jobs. Managers can view team statistics, track active job counts, and reset team member passwords when needed.

Manager Access Required
This page documents features only available to Manager and Admin users. If you don't see the My Team item under the Management section in the sidebar, contact your system administrator to request the Manager role.

Manager Page Overview

The manager page at /web/manager is called My Team and provides a view of all users you manage. It auto-refreshes every 10 seconds.

My Team Page Layout
The page shows three main areas: team statistics at the top, a Team Members table in the middle, and a Team Recent Jobs section below. All sections auto-refresh via HTMX at the manager interval (10 seconds).

Manager Permissions Summary

  • View all jobs from managed team members on the Jobs page
  • Cancel team member jobs
  • Submit restore jobs on behalf of managed users (via /web/restore?for_user=true)
  • Reset passwords for managed users
  • Cannot modify team membership (Admin only)
  • Cannot access admin-only pages

Team Statistics

A row of statistics cards at the top of the manager page provides a quick snapshot of your team's current activity.

Statistic Description
Managed Users Total number of team members you manage
Team Queued Jobs in queued state across the whole team
Team Running Jobs actively running across the whole team
Team Completed Jobs completed today across the whole team
Team Failed Failed jobs today across the whole team

Team Members Table

The Team Members table displays all users under your management. The table supports column sorting and text filtering, and loads lazily (LazyTable with virtual scroll).

Team Members Table Columns
The table shows each team member's username, user code, active job count, account status (active), and a password reset pending indicator. It does not show account states such as Inactive or Locked — those are admin-only attributes managed from the Admin Users page.

Team Member Table Columns

Column Description
Username The team member's login username
Code The short user code used as a database name prefix
Active Jobs Number of jobs currently in an active state (queued, running, canceling, or deployed) for this user
Status Account status — always active for team members visible to a manager
Password Reset Shows pending if the user has an outstanding forced password reset; otherwise blank
No Inactive or Locked States Here
The Team Members table does not show Inactive or Locked account states. Disabling, re-enabling, or unlocking accounts are admin-only actions available from /web/admin/users. The manager view shows only active team members.

Team Recent Jobs

Below the Team Members table, the manager page shows the last 5 jobs across all team members. This gives you a quick view of recent team activity without navigating to the full Jobs page.

Full Team Job Visibility
To view all team jobs (not just the most recent 5), navigate to the Jobs page at /web/jobs. As a manager, you can see all jobs from your managed users on that page.

Manager Actions

In addition to monitoring, managers have several actions available that extend beyond a regular user's capabilities.

Submitting Jobs for Team Members

Managers can submit restore jobs on behalf of a managed user. The job will appear in that user's job list as if they submitted it themselves.

  1. Navigate to /web/restore?for_user=true (or use the New Restore link — a user selector appears at the top for managers)
  2. Select the team member from the user selector that appears above the target configuration section
  3. Complete the restore form as normal
  4. Click Queue Restore — the job is submitted on behalf of the selected user

Canceling Team Member Jobs

Managers can cancel jobs submitted by any of their managed team members. Navigate to the Jobs page, find the team member's job, and use the cancel action.

Resetting Team Member Passwords

Managers can reset passwords for team members who have forgotten their credentials or need their account recovered. This flags the account for a forced password change on next login.

How the Reset Flow Works
When you reset a team member's password, the system generates new credentials and flags the account with a forced password change. The next time the user logs in, they are redirected to /web/change-password and cannot access the application until they set a new permanent password.

Reset Password Steps

  1. Navigate to the Team Members table on the My Team page (/web/manager)
  2. Find the user who needs a password reset
  3. Click the Reset Password action for that user (available in the row or via the actions menu)
  4. Confirm the action in the dialog
  5. The system flags the account — the Password Reset column for that user will show pending
  6. Provide the user with new temporary credentials via a secure out-of-band channel
  7. On their next login, they are automatically redirected to /web/change-password to set a permanent password
Security Note
Password resets are logged for audit purposes. Always verify the team member's identity through an out-of-band channel before resetting their password. Communicate new credentials securely (phone call or in-person, not email).
