Manager Interface

The Manager interface provides tools for managing your team members and monitoring their activity. Managers can view team statistics, reset passwords, and assign temporary credentials.

Manager Access Required
This page documents features only available to Manager users. If you don't see the Manager menu item, contact your system administrator to request the Manager role.

Manager Dashboard Overview

The manager dashboard provides a comprehensive view of your team's activity and key statistics. From here, you can quickly assess team performance and access management functions.

Dashboard Components

  • Team Statistics — Overview of active team members and their job counts
  • Activity Summary — Recent activity from all team members
  • Quick Actions — Fast access to common manager tasks
  • Status Indicators — Real-time status of team member accounts

Team Members

The team members section displays all users under your management. You can view their current status, recent activity, and perform account management tasks.

Team Member Information

Column Description
Name The team member's display name and username
Status Account status (Active, Inactive, Locked)
Last Activity When the user last performed an action
Jobs Total number of jobs submitted by this user
Actions Management actions available for this user

Status Indicators

  • ● Active — User can log in and submit jobs normally
  • ● Inactive — User account is disabled but not deleted
  • ● Locked — User is locked out due to failed login attempts

Resetting Passwords

Managers can reset passwords for team members who have forgotten their credentials or need their account recovered. This triggers a password reset flow for the selected user.

Reset Password Steps

  1. Navigate to the Team Members section
  2. Find the user who needs a password reset
  3. Click the Actions menu (⋮) for that user
  4. Select Reset Password
  5. Confirm the action in the dialog
  6. The user will receive instructions to set a new password
Security Note
Password resets are logged for audit purposes. Ensure you verify the team member's identity through an out-of-band channel before resetting their password.

Temporary Passwords

For immediate access needs, managers can assign a temporary password that the user must change on their next login. This is useful for onboarding new team members or providing urgent access recovery.

Assigning a Temporary Password

  1. Navigate to the Team Members section
  2. Click the Actions menu (⋮) for the target user
  3. Select Assign Temporary Password
  4. A secure temporary password will be generated automatically
  5. Click Copy to copy the password to your clipboard
  6. Securely communicate the password to the team member
Automatic Expiration
Temporary passwords expire after 24 hours if not used. The user will be required to set a new permanent password upon first login with the temporary credentials.

Best Practices

  • Verify Identity — Always confirm the user's identity before issuing credentials
  • Secure Delivery — Use secure channels (phone call, in-person) to share temporary passwords
  • Prompt Usage — Encourage users to log in and change their password immediately
  • Documentation — Note the reason for temporary password assignment if required by policy
