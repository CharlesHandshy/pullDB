Admin Panel

The Admin Panel provides system-wide configuration and management capabilities. Only users with the Admin role can access these features.

Admin Access Required
This page documents features only available to Admin users. If you don't see the Admin menu item, contact your system administrator to request access.

Overview

The Admin Panel is your central hub for managing the pullDB system. It's organized into tabs for different administrative functions.

Users

Create, edit, and manage user accounts.

Hosts

Configure MySQL hosts for restores.

API Keys

View and manage all system API keys.

Settings

System-wide configuration options.

User Management

The Users tab lets you manage all user accounts in the system.

User List

The user table displays all accounts with their key properties:

Column Description
Username Unique login identifier
Role User, Manager, or Admin
Status Active or Disabled
Hosts Number of authorized hosts
Last Login Most recent authentication
Actions Edit, Reset Password, Disable

Creating Users

Click "Create User" to add a new account. Required fields:

  • Username — Unique identifier (alphanumeric, no spaces)
  • Password — Initial password (user should change on first login)
  • Role — Permission level
  • Hosts — Which hosts the user can restore to

Editing Users

Click the Edit button to modify a user's settings. You can change:

  • Role (promote/demote)
  • Host authorizations (add/remove)
  • Account status (enable/disable)
Reset Password
Use the "Reset Password" action to generate a new temporary password for a user. The password will be displayed once—make sure to communicate it securely.
Cannot Delete Users
Users cannot be deleted to maintain audit trail integrity. Disable accounts instead. Disabled users cannot log in but their job history is preserved.

Host Management

The Hosts tab configures which MySQL servers are available as restore targets.

Host List

Column Description
Name Friendly name for the host
Hostname DNS name or IP address
Port MySQL port (default 3306)
Status Connection status indicator
Users Number of users with access

Adding Hosts

Click "Add Host" to configure a new MySQL server:

  • Name — Display name (e.g., "Staging MySQL")
  • Hostname — Server address
  • Port — MySQL port
  • Admin User — MySQL user with CREATE DATABASE privileges
  • Admin Password — Password for the admin user

Testing Connection

Use the "Test Connection" button to verify connectivity before saving. The test checks:

  • Network connectivity to the host
  • MySQL authentication
  • Required privileges (CREATE, DROP, etc.)
Credential Security
Host credentials are encrypted at rest. However, the MySQL admin user should have minimal required privileges and preferably only allow connections from the pullDB server.

API Key Management

The API Keys tab shows all API keys in the system across all users.

Global Key View

Unlike the user profile page (which shows only personal keys), this view shows:

  • All keys from all users
  • Which user owns each key
  • Usage statistics across the system
  • Keys that haven't been used recently

Key Actions

Action Description
View Details See full key properties and usage history
Revoke Immediately invalidate a key
Filter by User Show keys for a specific user
Filter by Host Show keys for a specific host
Audit Trail
When you revoke a key, the action is logged with your username and timestamp. The user who owns the key will be notified by email (if configured).

System Settings

The Settings tab contains global configuration options.

Available Settings

Setting Description Default
Max Concurrent Jobs Maximum jobs running simultaneously 3
Job Timeout Maximum time for a single job 2 hours
Retention Days How long to keep job history 90 days
Default Database Prefix Prefix for restored database names _staging
S3 Bucket Backup source bucket (configured at install)
Restart Required
Some settings (like Max Concurrent Jobs) require a worker restart to take effect. The UI will indicate when a restart is needed.

Maintenance

The Maintenance section provides tools for system upkeep.

Maintenance Actions

Cleanup

Remove old job records and temporary files. Respects retention settings.

Queue Clear

Cancel all queued jobs. Running jobs are not affected.

Force Reset

Reset stuck jobs to failed state. Use if jobs are hung indefinitely.

System Health

The health panel shows real-time system status:

  • Worker Status — Is the worker service running and healthy?
  • Queue Depth — Number of jobs waiting to be processed
  • Disk Space — Available space on the restore server
  • S3 Connectivity — Can we reach the backup bucket?
  • Database Connections — Status of MySQL host connections

Logs

Access system logs directly from the UI:

  • API Logs — HTTP request/response logging
  • Worker Logs — Job processing details
  • Auth Logs — Login attempts and API key usage
