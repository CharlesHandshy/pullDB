Download Issues

Symptoms: Job fails in download phase with S3 access error.

Causes:

  • AWS credentials expired or missing
  • IAM role doesn't have S3 read permissions
  • Wrong S3 bucket or path configured

Solutions:

  1. Check your AWS credentials are current:
    $ aws sts get-caller-identity
  2. Verify S3 access:
    $ aws s3 ls s3://your-bucket/backups/
  3. Contact your admin to verify IAM permissions

Symptoms: Job fails immediately with "backup not found" message.

Causes:

  • Customer ID misspelled
  • Backup doesn't exist for requested date
  • Backup was deleted or moved

Solutions:

  1. Verify available backups:
    $ pulldb list <customer>
  2. Double-check customer ID spelling
  3. Try without date= to get latest backup

Symptoms: Download starts but times out before completing.

Causes:

  • Network connectivity issues
  • Very large backup file
  • S3 throttling

Solutions:

  1. Check network connectivity
  2. For large backups, expect longer times (check pulldb events for progress)
  3. Retry the job - transient network issues often resolve

Restore Issues

Symptoms: Job fails in myloader phase with CREATE DATABASE error.

Causes:

  • MySQL staging user lacks CREATE privilege
  • Database name collision

Solutions:

  1. Contact admin to verify staging user permissions
  2. Check for orphaned staging databases:
    $ pulldb-admin cleanup --dry-run

Symptoms: Job fails in atomic_rename phase because target database exists.

Solution: Add overwrite to replace the existing database:

$ pulldb restore acme overwrite
Warning: overwrite will delete the existing database. Make sure you don't need it!

Symptoms: Job fails in post_sql phase.

Causes:

  • Schema changed and script references missing columns
  • Script has SQL syntax errors
  • Large UPDATE times out

Solutions:

  1. Check job events for the exact error:
    $ pulldb events <job_id>
  2. Contact admin to review/update post-SQL scripts

Permission Issues

Symptoms: Can't submit jobs or run commands.

Causes:

  • Username not registered in system
  • User account disabled

Solutions:

  1. Verify your username is correct
  2. Contact admin to enable your account

Symptoms: Job completes but you can't access the database.

Causes:

  • MySQL user doesn't have GRANT on the restored database
  • Host restriction on MySQL user

Solutions:

  1. Verify database exists:
    $ mysql -e "SHOW DATABASES LIKE '%yourcode%'"
  2. Contact admin to verify MySQL permissions

Connection Issues

Symptoms: CLI commands fail with connection errors.

Causes:

  • pullDB service not running
  • Firewall blocking port
  • Wrong API URL configured

Solutions:

  1. Check your configuration:
    $ cat ~/.pulldb/config
  2. Verify service is reachable
  3. Contact admin if service is down

Job Stuck

Symptoms: Job stays in QUEUED for more than a few minutes.

Causes:

  • No workers running
  • All workers busy with other jobs
  • Worker crashed

Solutions:

  1. Check how many jobs are ahead of you:
    $ pulldb status
  2. If stuck for >5 minutes with no queue, contact admin

Symptoms: Job shows RUNNING but no progress for a long time.

Solutions:

  1. Check events for the current phase:
    $ pulldb events <job_id> --follow
  2. download - Large backup, check progress %
  3. myloader - Large database, can take 10+ minutes
  4. If truly stuck, cancel and retry:
    $ pulldb cancel <job_id>

Slow Restore

Expected restore times
Small databases (<1GB): 2-5 minutes
Medium databases (1-5GB): 5-15 minutes
Large databases (5-20GB): 15-45 minutes

If your restore is slower than expected:

  • Check which phase is taking longest with pulldb events
  • Downloading slow: Network or S3 throughput issue
  • Restoring slow: MySQL I/O or table complexity (indexes)
  • Contact admin if consistently slower than expected

Disk Space Issues

Symptoms: Job fails with disk space errors.

Space requirements:

  • Download: Size of compressed backup
  • Extract: 2-3x compressed size
  • Total: ~4x compressed backup size

Solutions:

  1. Check available space:
    $ df -h
  2. Clean up old files or contact admin
  3. Request admin run cleanup for orphaned files

Still Need Help?

If you can't find a solution here:

  • Run pulldb events <job_id> and share the output
  • Note the exact error message
  • Contact your pullDB administrator
