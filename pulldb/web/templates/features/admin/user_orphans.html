User Orphan Databases

Destructive Operation
These databases belong to users who no longer exist in the system. The user_code prefix in the database name does not match any active user. Deletion cannot be undone.
User orphans are databases where the 6-character user_code prefix (e.g., jdoesacme) doesn't match any user in the system. This typically happens when users are deleted but their databases remain.
